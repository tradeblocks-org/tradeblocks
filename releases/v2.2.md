# TradeBlocks MCP Server 2.2 — Market Data Provider Abstraction + Trade Replay + Greeks

Trade replay with minute-level greeks, exit trigger analysis across your whole portfolio, and a normalized VIX data model that supports any tenor. Massive.com is the default data provider — all replay and analysis tools work with locally cached bar data too. If you have a Massive subscription, you get one-click market data import instead of manual CSV exports. **New in beta 2:** a provider abstraction layer lets you plug in alternative data sources (ThetaData, custom APIs). **New in beta 5-7:** profit action triggers with partial closes, directional delta exits, and bid/ask mid mark pricing.

## Market Data Provider Abstraction (beta 2)

The `import_from_api` tool replaces the former `import_from_massive`. Point it at a ticker and date range, and it fetches daily OHLCV, intraday bars (1m/5m/15m/1h), or option minute bars from the configured data provider. The default provider is Massive.com (Polygon). Set `MARKET_DATA_PROVIDER` env var to switch providers. The existing `import_market_csv` workflow continues to work exactly as before.

All providers implement the same `MarketDataProvider` interface and normalize to the same `BarRow` and `OptionContract` types. Downstream tools (replay, exit analysis, enrichment) work identically regardless of provider. See the [Market Data guide](https://github.com/davidromeo/tradeblocks/blob/feature/massive-integration/docs/market-data.md#building-a-custom-provider) for how to build your own provider adapter.

For VIX context, `target_table: "context"` is a convenience shorthand that fetches VIX + VIX9D + VIX3M in one call and triggers enrichment automatically.

**Setup:** Set `MASSIVE_API_KEY` as an environment variable (or the appropriate key for your provider). See the [Getting Started guide](https://github.com/davidromeo/tradeblocks/blob/feature/massive-integration/docs/getting-started.md#environment-variables) and [Market Data guide](https://github.com/davidromeo/tradeblocks/blob/feature/massive-integration/docs/market-data.md) for configuration details. Everything else works exactly as before without it.

## Trade Replay

The `replay_trade` tool resolves your tradelog leg descriptions to OCC tickers (e.g., `SPY 450P/445P` → `O:SPY250117P00450000`) and combines minute-level option bars into a strategy P&L path with MFE and MAE. Two ways to use it: pass explicit legs, or point it at a block_id + trade_index from your existing tradelogs.

Bar data comes from `market.intraday` cache first. On a cache miss, it fetches from the configured data provider (default: Massive.com). After the first fetch, bars are persisted locally — all subsequent replays of the same trade are instant with zero API calls. You can also pre-load bars via `import_market_csv` with intraday data and skip the API entirely.

Three output formats: `full` (every minute-level data point), `sampled` (default — ~25-30 points at ~15min intervals), and `summary` (key metrics only). Use `close_at: "expiry"` to see what would have happened if you held through expiration instead of your actual exit.

## Greeks Engine

Every replay point now includes per-leg greeks (delta, gamma, theta, vega, IV) and net position greeks. Black-Scholes with Newton-Raphson IV solver handles regular options. For 0DTE options where BS gamma explodes, a Bachelier normal model kicks in automatically below 0.1 days to expiry. Every greeks result includes a `model` field (`'bs'` or `'bachelier'`) so you know which one computed it.

## Exit Trigger Analysis

Three new tools for understanding when and why to exit trades. All use the same cache-first bar loading as replay — no Massive subscription required if you have bars in `market.intraday`.

- `analyze_exit_triggers` — evaluates 14 trigger types against a single trade's replay path. Supports profitTarget, stopLoss, trailingStop, DTE/DIT exits, clock time, underlying moves, position delta, per-leg delta, VIX moves, VIX9D/VIX ratio, and stop-loss ratio triggers. Shows which trigger fires first and compares against your actual exit.
- `decompose_greeks` — breaks down P&L into delta, gamma, theta, vega, charm, vanna, and residual contributions between any two points. Uses model-based attribution by default with automatic numerical fallback (finite-difference repricing) when residual exceeds 80%. For calendar strategies, includes per-leg-group vega attribution showing front vs back month IV divergence.
- `batch_exit_analysis` — the portfolio-level version. Replays matching trades from a block, evaluates a candidate exit policy against each one, and returns aggregate stats directly comparable to `get_statistics`: win rate, avg P&L, profit factor, payoff ratio, Sharpe, Sortino, Calmar, max drawdown, win/loss streaks, skewness, kurtosis, and more. Per-trigger attribution tells you which trigger is doing the heavy lifting.

Percentage-based thresholds (`unit: 'percent'`) are supported for profit targets and stop losses — set a 50% profit target instead of calculating dollar amounts. Leg groups let you set different triggers for each structure in a multi-leg position (e.g., close the put spread at 50% but let the call spread ride).

## Live Options Snapshot

`get_option_snapshot` fetches real-time option chain data (greeks, IV, open interest, quotes) from Massive's snapshot endpoint. Auto-paginates through the full chain. When Massive returns empty greeks for a contract, Black-Scholes fills them in as a fallback — each contract shows `greeks_source: 'massive'` or `'computed'` so you know which is which.

## Normalized VIX Term Structure

VIX data has been restructured. Instead of hardcoded VIX_Close, VIX9D_Close, VIX3M_Close columns in `market.context`, VIX tenors are now regular ticker rows in `market.daily` with `ivr` and `ivp` columns. Cross-ticker derived fields (Vol_Regime, Term_Structure_State, Trend_Direction) live in a new `market._context_derived` table.

The enrichment pipeline discovers VIX tickers dynamically — add VIX1D or VIX6M and it computes IVR/IVP automatically. All query builders, tool descriptions, and example queries have been updated. `describe_database` now returns available VIX tenors with ready-to-use query patterns.

Existing databases migrate automatically on first startup. `market.context` is preserved for backward compatibility.

## Bid/Ask Mid Mark Pricing (beta 7)

Replay and greeks calculations now use bid/ask midpoint as the mark price when quote data is available, falling back to HL2 (high+low)/2 when it's not. This produces more accurate marks for thinly traded options where high/low trade prints can be outliers that never coexisted in the market.

For Massive.com users with quotes tier access, option intraday bar fetches automatically try the `/v3/quotes` endpoint to populate bid/ask. If the endpoint isn't available on your tier, the fetch silently degrades — bars retain HL2 pricing with zero user impact. ThetaData and custom providers can populate bid/ask directly on import.

The `market.intraday` table now includes `bid` and `ask` columns. Existing databases migrate automatically.

## Profit Action Triggers (beta 5)

The former "MFE ladder stop" trigger has been renamed to `profitAction` to match Option Omega terminology. New capabilities:

- **Partial position closing** via `closeAllocationPct` — close 50% at the first profit target and let the rest ride. P&L aggregation across partial closes is handled automatically in `batch_exit_analysis`.
- **Directional delta exits** — `perLegDelta` and `positionDelta` triggers now support `legIndex`, `exitAbove`, and `exitBelow` fields for directional exit rules (e.g., "close when short put delta exceeds -0.30").

## ChatGPT Compatibility (beta 3)

MCP tool schemas now use `z.array` instead of `z.tuple` for fixed-length array parameters. ChatGPT's MCP client rejects `prefixItems` (the JSON Schema representation of tuples), so this change ensures all TradeBlocks tools work across both Claude and ChatGPT.

## SQL Confirm Gate

`run_sql` now requires `confirm: true` for all mutating operations (DELETE, UPDATE, INSERT, CREATE, ALTER, DROP, TRUNCATE). Without it, DELETE and UPDATE return a preview of how many rows would be affected. Prevents accidental data loss when exploring via SQL.

## New Tools

| Tool                    | What it does                                                         | Requires Massive?                      |
| ----------------------- | -------------------------------------------------------------------- | -------------------------------------- |
| `import_from_api`       | Import daily, intraday, or option bars from configured data provider | Yes (default: Massive.com)             |
| `get_option_snapshot`   | Live option chain with greeks, IV, and open interest                 | Yes                                    |
| `replay_trade`          | Minute-level P&L path with MFE/MAE and per-leg greeks                | No — uses cached bars, fetches on miss |
| `analyze_exit_triggers` | Evaluate 14 exit trigger types against a trade's replay              | No — uses cached bars, fetches on miss |
| `decompose_greeks`      | P&L decomposition into greek factor contributions                    | No — uses cached bars, fetches on miss |
| `batch_exit_analysis`   | Test exit policies across entire blocks with aggregate stats         | No — uses cached bars, fetches on miss |

## Breaking Changes

- `import_from_massive` renamed to `import_from_api` (beta 2). The tool behavior is identical — only the name changed to reflect the provider abstraction. Update any saved prompts or workflows that reference the old name.
