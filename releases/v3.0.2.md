# TradeBlocks MCP Server 3.0.2 — Parallel implied-volatility solve

A patch release that speeds up market-data ingestion. Output is byte-identical to 3.0.1 — this is purely a performance change.

## Performance: implied-volatility solve runs across worker threads

Minute-level option-quote ingestion solves an implied volatility (and first-order greeks) for every quote row. That solve previously ran single-threaded on the main event loop, where it dominated ingest time on dense option chains.

The solve now fans out across a `worker_threads` pool, so multi-core hosts ingest dense chains substantially faster. The work is unchanged — the parallel path calls the same solver as before, so deltas, gammas, thetas, vegas, implied vols, and all provenance fields are identical to the single-threaded result. A real-trading-day parity check across 804,715 greek comparisons found a maximum difference of exactly zero.

The pool is on by default and auto-degrades to the inline solver on single-core hosts and small batches. Set `TRADEBLOCKS_IV_WORKERS=0` (or `off`) to force the inline path. A worker that fails to spawn fails loudly rather than silently falling back to a wrong result.

No public API, tool, or output changes.
