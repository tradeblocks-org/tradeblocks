# TradeBlocks MCP Server 3.4.0 — Content-addressed market-data provenance

This release adds a public foundation for exact-byte provenance on canonical market-data Parquet writes.

## New public API

The `tradeblocks-mcp/market/provenance` export provides:

- versioned canonical JSON encoding with NFC normalization, integer-only numeric semantics, and exact-byte SHA-256 content addresses;
- a crash-safe, immutable, no-replace canonical JSON object store;
- versioned partition commit receipts with schema revision, logical coverage, quality counts, and append/repair classification;
- a file-backed receipt store with serialized partition-head updates, dead-owner lock recovery, verification, and orphan detection; and
- attempt-scoped receipt capture that leaves existing market store method contracts unchanged.

Canonical market-dataset writers now supply stable dataset, partition, and relative-path identities to the provenance layer. Exact-byte hashing and receipt persistence remain opt-in: they run only inside a partition commit attempt. Existing callers outside an attempt retain their prior write behavior and avoid the additional hashing work.

If a Parquet file is atomically published but receipt persistence fails, the write fails with an explicit orphan error while leaving the published file available for inspection and reconciliation.
