# TradeBlocks MCP Server 3.3.1 — Distribution consolidated on npm; MCPB bundle discontinued

A packaging release. The `.mcpb` (MCP Bundle / Desktop Extension) distribution is removed; the server is now installed exclusively via npm (npx), Docker, or from source. There are no runtime, tool-surface, or public-API changes in the server itself.

This release also delivers the library work from the 3.1.0–3.3.0 notes to npm for the first time — see "Version reconciliation" below.

## Change: the `.mcpb` bundle is no longer published

Previous releases attached a `tradeblocks-*.mcpb` asset intended for one-click install in Claude Desktop. That bundle could not run: it shipped the compiled server code with **no bundled dependencies**, so the server crashed at startup on the first unresolved import (the native database binding was simply the first of many). Native bindings cannot be portably bundled into an `.mcpb`, and the ABI must match the host runtime — constraints the bundle format does not satisfy for this server.

The npm distribution has none of these problems: `npm`/`npx` resolves every dependency — including the native binding — against your own runtime at install time, so it is built for your exact platform and Node ABI. It has always been the working path.

### How to install now

```bash
# All platforms — resolves dependencies for your runtime
npx tradeblocks-mcp ~/path/to/backtests
```

For Claude Desktop, add the server to `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "tradeblocks": {
      "command": "npx",
      "args": ["tradeblocks-mcp", "~/path/to/backtests"]
    }
  }
}
```

Docker and from-source install are unchanged. See the package README for per-platform configuration.

## Version reconciliation

The 3.1.0, 3.2.0, and 3.3.0 releases bumped the repository version and published release notes, but did not bump the published package version — which the release pipeline keys its npm publish on. As a result those versions never reached npm, and the accumulated library changes they describe sat unpublished. 3.3.1 realigns the published package with the repository version, so this release carries that accumulated library work to npm as well.

To prevent a recurrence, the release pipeline now fails fast when the repository and published-package versions diverge, instead of silently skipping the publish.
